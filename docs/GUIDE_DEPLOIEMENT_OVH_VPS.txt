GUIDE COMPLET DEPLOIEMENT - CHATBOT JURIDIQUE (OVH VPS)
Version: 2026-02-26
Public: Windows + VPS OVH Ubuntu
Format: pas a pas, tres detaille, copy/paste

========================================================================
0) OBJECTIF
========================================================================
Ce guide te permet de:
1. Deployer ton backend RAG sur ton VPS OVH.
2. Garder toutes les fonctionnalites locales en ligne:
   - chat
   - RAG
   - bibliotheque PDF
   - ouverture PDF en lecture (pas telechargement force)
   - sources workspace
3. Exposer le backend proprement (Nginx + SSL optionnel).
4. Connecter le frontend web et l'app mobile au backend VPS.

IMPORTANT:
- Le backend attend les fichiers RAG deja construits dans data/index.
- Le backend attend les PDF dans le dossier droit donnees.
- Sans ces fichiers, /ready peut etre en erreur ou le RAG sera incomplet.

========================================================================
1) ARCHITECTURE CIBLE
========================================================================
[Frontend Web] ----HTTP/HTTPS----\
                                  > [Nginx] -> [Uvicorn FastAPI backend]
[App mobile]  ----HTTP/HTTPS----/                     |
                                                      -> data/index (FAISS)
                                                      -> droit donnees (PDF)

Sur VPS:
- Service Python: backend.app.main:app
- Port interne app: 8000
- Port externe: 80/443 via Nginx

5) TRANSFERER LE PROJET SUR LE VPS
========================================================================
Tu as 2 methodes. Prends UNE methode.

METHODE A (recommandee): rsync depuis ton PC
Prerequis: utiliser Git Bash (ou WSL) pour rsync.

Depuis ton PC, dans le dossier parent du projet:
1. cd "C:/Users/abtru/Desktop"
2. rsync -avz --delete \
   --exclude ".git" \

1. ssh ubuntu@51.77.211.7
2. cd /home/ubuntu

ATTENTION:
- Si data/index ou droit donnees ne sont PAS dans git, utilise METHODE A.

6) BOOTSTRAP BACKEND SUR VPS
========================================================================
Connecte-toi au VPS:
1. ssh ubuntu@51.77.211.7
2. cd /home/ubuntu/chatbot-juridique

8) CONFIGURER .env PRODUCTION
========================================================================
Depuis /home/ubuntu/chatbot-juridique:
1. cp .env.example .env
2. nano .env

Configuration minimale conseillee (NVIDIA):

APP_ENV=production
LLM_PROVIDER=nvidia
NVIDIA_API_KEY=TA_CLE_REELLE
NVIDIA_BASE_URL=https://integrate.api.nvidia.com/v1
NVIDIA_MODEL=deepseek-ai/deepseek-v3.2

RAG_ENABLED=true
RAG_INDEX_DIR=data/index
LEGAL_DATA_DIR=droit donnees

SPEECH_ENABLED=false
RAG_RERANKER_ENABLED=false

ALLOWED_ORIGINS=http://localhost:3000,http://127.0.0.1:3000,http://localhost:7410,http://127.0.0.1:7410,https://TON_FRONTEND_WEB
TRUSTED_HOSTS=localhost,127.0.0.1,51.77.211.7,TON_DOMAINE_BACKEND

Si tu utilises OLLAMA a la place:
APP_ENV=production
LLM_PROVIDER=ollama
OLLAMA_BASE_URL=http://127.0.0.1:11434/v1
OLLAMA_MODEL=deepseek-v3.1:671b-cloud
OLLAMA_API_KEY=ollama

IMPORTANT:
- Si tu laisses NVIDIA_API_KEY vide avec LLM_PROVIDER=nvidia, le chat echouera.
- Pour mobile + web + local, garde ALLOWED_ORIGINS large pendant tests.

- Content-Type: application/pdf

10) PASSER EN SERVICE SYSTEMD
========================================================================
Le projet contient:
- deploy/systemd/juridique-backend.service

10.1 Adapter le fichier si besoin:
1. nano deploy/systemd/juridique-backend.service
2. Verifie:
   - User=ubuntu
   - WorkingDirectory=/home/ubuntu/chatbot-juridique
   - ExecStart=/home/ubuntu/chatbot-juridique/scripts/vps/run_backend.sh

10.2 Installer le service:
1. sudo cp deploy/systemd/juridique-backend.service /etc/systemd/system/
2. sudo systemctl daemon-reload
3. sudo systemctl enable juridique-backend
4. sudo systemctl start juridique-backend
5. sudo systemctl status juridique-backend --no-pager

10.3 Logs en direct:
1. journalctl -u juridique-backend -f

========================================================================
18) TROUBLESHOOTING RAPIDE
========================================================================
Cas A - 502 Bad Gateway (Nginx):
1. sudo systemctl status juridique-backend --no-pager
2. journalctl -u juridique-backend -n 200 --no-pager
3. curl http://127.0.0.1:8000/health
4. si backend KO, corrige .env / deps / index manquant

Cas B - /ready en 503:
Cause frequente:
- index.faiss ou meta.jsonl manquant
Action:
1. ls -lh data/index/
2. recopier index

Cas C - CORS bloque dans navigateur:
1. verifier ALLOWED_ORIGINS dans .env
2. inclure exactement l'origine frontend (schema + domaine + port)
3. restart service

Cas D - PDF ne s'ouvre pas et telecharge:
1. tester endpoint /view avec curl -I
2. verifier Content-Disposition inline
3. verifier frontend/mobile utilise bien /view (pas /download)

Cas E - Mobile n'atteint pas backend:
1. verifier --dart-define BACKEND_URL
2. verifier que l'URL est accessible depuis le telephone (reseau)
3. si HTTP non-SSL externe, certains appareils le bloquent

Cas F - Reponses RAG lentes:
1. garder UVICORN_WORKERS=1 sur 8GB RAM
2. SPEECH_ENABLED=false
3. RAG_RERANKER_ENABLED=false (ou ajuster plus tard)
